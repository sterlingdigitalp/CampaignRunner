# Runtime 2.0 Roadmap

## Priority 1: Execution Scale

- Dependency-aware parallel execution for independent tasks.
- Checkpoint execution as first-class runtime stages.
- Queue-based execution controls for long campaigns.

## Priority 2: Verification Depth

- Profile-specific verification suites.
- Documentation consistency verifiers.
- Expected-output comparison against workspace files.
- Benchmark-level verification scoring.

## Priority 3: Repair Intelligence

- Adaptive repair by protocol category.
- Persistent protocol learning across benchmarks.
- Safer partial acceptance for independent file blocks.
- Model-specific repair prompt tuning.

## Priority 4: Platform Integration

- BuilderBoard integration.
- Knowledge Service integration.
- Multiple model routing.
- Remote workers and distributed execution.

## Priority 5: Analytics

- Campaign analytics across benchmark history.
- Repair category trends.
- Task-time prediction.
- Execution optimization recommendations.

## Benchmark Evidence Basis

Benchmark 001 exposed repair specificity as the key bottleneck. Benchmark 002 showed targeted repair and replayable telemetry are the highest leverage foundation. Runtime 2.0 should build on that evidence rather than replacing the Runtime 1.0 execution model.
